The Model Context Protocol (MCP) server connects your AI applications to browser automation. This allows AI agents in tools like Claude Desktop, Cursor, or Windsurf to fill out forms, download files, research information on the web, and more.

Skyvern's MCP integration works in two modes:

- **Skyvern Cloud** — Connect to [app.skyvern.com](https://app.skyvern.com) with your API key
- **Local Skyvern Server** — Run everything locally with your own LLM

---

## Step 1: Install the SDK

<Note>
Skyvern requires Python 3.11, 3.12, or 3.13.
</Note>

```bash
pip install skyvern
```

---

## Step 2: Run the setup wizard

The interactive wizard configures your MCP client (Claude Desktop, Cursor, Windsurf) and sets up credentials.

```bash
skyvern init
```

The wizard will:
1. Ask whether to use Skyvern Cloud or local mode
2. Configure your LLM provider (if local)
3. Set up your MCP client configuration automatically

---

## Step 3: Start the server (local mode only)

If you chose local mode, start the Skyvern server:

```bash
skyvern run server
```

Skip this step if you're using Skyvern Cloud.

## Manual configuration

For other MCP-enabled applications, add this to your MCP configuration file:

```json
{
  "mcpServers": {
    "Skyvern": {
      "env": {
        "SKYVERN_BASE_URL": "https://api.skyvern.com",
        "SKYVERN_API_KEY": "YOUR_SKYVERN_API_KEY"
      },
      "command": "PATH_TO_PYTHON",
      "args": [
        "-m",
        "skyvern",
        "run",
        "mcp"
      ]
    }
  }
}
```

For local mode, set `SKYVERN_BASE_URL` to `http://localhost:8000`. Find your local `SKYVERN_API_KEY` in the `.env` file after running `skyvern init`.
