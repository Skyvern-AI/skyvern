Skyvern routes browser traffic through residential proxies so every run looks like it's coming from a real consumer ISP in the country you choose. Locale and timezone auto-match the proxy region, so you never end up with a mismatched fingerprint.

---

## Coverage

- **21 countries** of residential coverage — Americas (US, CA, MX, AR, BR), Europe (GB, IE, NL, DE, FR, ES, IT, TR), Asia (IN, PH, KR, JP, SA), Africa (ZA), and Oceania (AU, NZ)
- **Static ISP IPs** via `RESIDENTIAL_ISP` for services that require a stable IP across a session
- **City- and state-level targeting** in the United States via `GeoTarget` — e.g. `{"country": "US", "subdivision": "CA", "city": "San Francisco"}`

---

## How it's applied

- **Per task**: pass `proxy_location` in the run request
- **Per workflow**: set a default on the workflow and override on any individual run
- **Per session**: lock a proxy to a [browser session](/optimization/browser-sessions) so every task in that session shares the same IP

---

## Why it matters

| Use case | How proxy & GEO help |
|----------|----------------------|
| Region-locked pricing, inventory, or UX | See exactly what a user in that country sees |
| Regulatory filings that require a local IP | Submit from inside the target jurisdiction |
| Reducing bot detection risk | Residential IPs look like real consumer traffic, not a datacenter |
| Session-bound services | `RESIDENTIAL_ISP` keeps the same IP for the whole session |
