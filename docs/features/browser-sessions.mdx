A Browser Session is a live browser instance that Skyvern keeps running between runs. Cookies, local storage, open tabs, and the current page all persist. Later tasks pick up exactly where the previous one stopped.

---

## What you can build

- **Back-to-back tasks** — log in with one task, search and extract with the next, no re-authentication in between
- **Human-in-the-loop flows** — pause for approval between steps without losing browser state
- **Real-time agents** — keep a browser open for hours while the agent responds to events
- **Task + workflow chaining** — run tasks and workflows through the same live browser instance

---

## Configuration

| Option | Range / Values |
|--------|----------------|
| Timeout | 5 minutes to 24 hours (default 60 min) |
| Proxy location | Any of Skyvern's [20 supported countries](/going-to-production/proxy-geolocation) |
| Browser type | Chrome or Edge |
| Extensions | Built-in ad-blocker, CAPTCHA solver |

Every session exposes a live VNC stream so you can watch — or take control of — the browser as it runs.

---

## Sessions vs. Profiles

Sessions are **live** browsers that bill while open. [Browser Profiles](/optimization/browser-profiles) are **saved snapshots** of browser state you reuse across days or weeks. You can create a profile from a finished session to save its authenticated state for later reuse.
