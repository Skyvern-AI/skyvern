When Skyvern's agent completes a task or workflow, it records the actions it took and generates reusable code from them. Later runs can execute that cached code directly, skipping LLM inference and screenshot analysis entirely.

---

## Why it matters

- **Faster** — cached runs skip screenshot analysis and LLM reasoning
- **Cheaper** — no inference means a fraction of the credit cost per run
- **Deterministic** — the same recorded actions execute the same way every time
- **AI fallback** — if the cached code fails on a page that changed, Skyvern falls back to the agent and regenerates

---

## How it works

1. **First run** — set `engine="skyvern-2.0"` when creating the task or workflow. The agent executes and Skyvern records the actions as generated code.
2. **Subsequent runs** — pass `run_with="code"` to execute the cached code directly, no agent needed.
3. **Self-healing** — if the cached code fails (layout change, new field, etc.), Skyvern re-runs with the agent and regenerates the cache.

<Note>
The SDK default engine is `skyvern-1.0`. To use code caching, explicitly set `engine="skyvern-2.0"`. In the Cloud UI, `skyvern-2.0` is the default.
</Note>

---

## What gets cached

- **Tasks** cache the action sequence the agent took (clicks, form fills, extractions)
- **Workflows** cache per block — each block that successfully executes gets its own cached script
- **Progressive caching** — workflows with conditionals cache branches as they execute: run 1 covers branch A, run 2 covers branch B, and so on. Previous caches are preserved
- **Not cached** — conditional evaluation blocks, wait blocks, and code blocks always run live

<Tip>
Pair cached code with [scheduled workflows](/multi-step-automations/scheduling-workflows) for cheap, reliable recurring runs — the first scheduled run builds the cache, and every subsequent run executes from it.
</Tip>
