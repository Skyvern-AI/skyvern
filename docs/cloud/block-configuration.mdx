Workflows are built from blocks. Each block performs one specific operation — navigating a page, extracting data, making an API call, or branching logic. This page covers every block type available in the [editor](/cloud/workflow-editor), grouped by category.

## Common fields

These fields appear on most blocks:

| Field | Description |
|-------|-------------|
| **Label** | Display name on the canvas. Also determines how downstream blocks reference this block's output (`{{ label_output }}`). |
| **Continue on Failure** | Allow the workflow to continue if this block fails |
| **Continue to Next Loop on Failure** | Only inside loops. Skip to the next iteration on failure. |

Browser-based blocks (Navigation, Action, Extraction, Login, File Download) share these additional fields:

| Field | Description |
|-------|-------------|
| **URL** | Starting URL. Leave blank to continue from the previous block's page. |
| **Engine** | AI engine: Skyvern 1.0 or Skyvern 2.0 |
| **Max Retries** | Retry attempts on failure (default: 0) |
| **Max Steps Override** | Cap the number of AI reasoning steps |
| **TOTP Identifier** | Link TOTP credentials for 2FA handling |
| **TOTP Verification URL** | Endpoint for fetching TOTP codes |
| **Error Code Mapping** | Map error conditions to custom error codes (JSON) |

## Browser Automation

### Browser Task v2

The primary block for browser automation. Accepts a natural-language prompt and autonomously navigates the browser to accomplish the goal. Supports multi-step objectives.

| Field | Type | Description |
|-------|------|-------------|
| **Prompt** | text | Natural language instructions for the AI |
| **URL** | text | Starting URL (optional) |
| **Max Steps** | number | Maximum AI steps (default: 25) |
| **Max Screenshot Scrolls** | number | Scrolls for post-action screenshots |
| **Disable Cache** | boolean | Skip cached script execution |

<Tip>
Browser Task v2 is the recommended block for most browser automation. Use it for anything from form filling to multi-page navigation.
</Tip>


### Navigation

AI-guided browser navigation toward a goal. Supports completion and termination criteria to control when the block finishes.

| Field | Type | Description |
|-------|------|-------------|
| **Navigation Goal** | text | What the AI should accomplish |
| **Complete Criterion** | text | Condition that signals the block is done |
| **Terminate Criterion** | text | Condition that signals the block should stop |
| **Allow Downloads** | boolean | Permit file downloads during navigation |
| **File Suffix** | text | Custom filename for downloaded files |
| **Include Action History** | boolean | Include action history in verification |

*Plus [common browser fields](#common-fields).*

### Action

Execute a single browser action. Best for precise, one-step operations like clicking a specific button or entering text in a field.

| Field | Type | Description |
|-------|------|-------------|
| **Navigation Goal** | text | Describe the single action to perform |
| **Allow Downloads** | boolean | Permit file downloads |
| **File Suffix** | text | Custom filename for downloads |

*Plus [common browser fields](#common-fields).*

### Login

Authenticate to a website using stored credentials. Pre-filled with a default navigation goal that handles common login flows including 2FA.

| Field | Type | Description |
|-------|------|-------------|
| **Navigation Goal** | text | Login instructions (pre-filled with a comprehensive default) |
| **Complete Criterion** | text | How to know login succeeded |
| **Terminate Criterion** | text | When to stop trying |

*Plus [common browser fields](#common-fields).*

### Go to URL

Navigate the browser directly to a specific URL without AI interaction.

| Field | Type | Description |
|-------|------|-------------|
| **URL** | text | **(Required)** The URL to navigate to. Supports [parameter references](/cloud/workflow-parameters#referencing-parameters-in-blocks). |

### Print Page

Print the current browser page to a PDF file. The PDF is saved as a downloadable artifact.

No additional fields beyond the [common fields](#common-fields).

## Data and Extraction

### Extraction

Extract structured data from the current page using AI.

| Field | Type | Description |
|-------|------|-------------|
| **Data Extraction Goal** | text | What data to extract |
| **Data Schema** | JSON | [JSON Schema](https://json-schema.org/) defining the output format |

*Plus [common browser fields](#common-fields).*

### Text Prompt

Send text to the LLM for processing without browser interaction. Useful for summarizing, transforming, or analyzing data between browser steps.

| Field | Type | Description |
|-------|------|-------------|
| **Prompt** | text | The text prompt to send to the LLM |
| **JSON Schema** | JSON | Expected output structure |
| **Input Parameters** | select | Parameters to include in the prompt context |

### File Parser

Parse file contents (PDF, CSV, Excel) and extract structured data.

| Field | Type | Description |
|-------|------|-------------|
| **File URL** | text | URL or parameter reference to the file |
| **JSON Schema** | JSON | Schema for the extracted output |

### PDF Parser

<Warning>
PDF Parser is deprecated. Use **File Parser** with a PDF file URL instead.
</Warning>

Specialized PDF text extraction. Internally uses the same engine as File Parser.

| Field | Type | Description |
|-------|------|-------------|
| **File URL** | text | URL or parameter reference to the PDF |
| **JSON Schema** | JSON | Schema for the extracted output |

## Control Flow

### Loop

Repeat a sequence of blocks for each item in a list. Child blocks are placed inside the loop on the canvas.

| Field | Type | Description |
|-------|------|-------------|
| **Loop Value** | text | The list to iterate over. Use a parameter reference (e.g., `{{ my_list }}`) or a natural language description (e.g., "Extract links of the top 5 posts") which triggers automatic extraction. |
| **Complete if Empty** | boolean | Mark the loop as complete if the list is empty |

Inside loop blocks, use these [reserved variables](/cloud/workflow-parameters#reserved-variables):
- `{{ current_value }}` — the current item
- `{{ current_index }}` — the iteration number (0-based)

<img src="/images/cloud/block-loop-config.png" alt="Loop block configuration" />

### Conditional

Branch the workflow based on conditions. Each branch has a Jinja2 expression that evaluates to true or false.

| Field | Type | Description |
|-------|------|-------------|
| **Branches** | list | One or more condition/target pairs. Each has a Jinja2 expression and a target block label. |
| **Default Branch** | auto | Fallback that executes when no condition matches |

**Example expression:**
```
{{ extraction_output.price > 100 }}
```

<img src="/images/cloud/block-conditional-config.png" alt="Conditional block configuration" />

### Validation

Assert conditions and halt the workflow on failure. Useful for checking that a previous block produced expected results before continuing.

| Field | Type | Description |
|-------|------|-------------|
| **Complete Criterion** | text | Condition that means validation passed |
| **Terminate Criterion** | text | Condition that means validation failed |
| **Error Code Mapping** | JSON | Custom error codes for failure reasons |
| **Input Parameters** | select | Parameters to evaluate |

### Code

Execute custom Python code. Input parameters are available as global variables. Top-level variables in your code become the block's output.

| Field | Type | Description |
|-------|------|-------------|
| **Code** | code editor | Python code to execute |
| **Input Parameters** | select | Parameters available as globals in the code |

### Wait

Pause workflow execution for a specified duration.

| Field | Type | Description |
|-------|------|-------------|
| **Wait in Seconds** | number | Seconds to wait (0–300) |

## Files

### File Download

Navigate the browser to download a file.

| Field | Type | Description |
|-------|------|-------------|
| **Navigation Goal** | text | Instructions for finding and downloading the file |
| **File Suffix** | text | Custom filename for the download |
| **Download Timeout** | number | Seconds to wait for the download to complete |

*Plus [common browser fields](#common-fields).*

### Cloud Storage Upload

Upload downloaded files to S3 or Azure Blob Storage.

| Field | Type | Description |
|-------|------|-------------|
| **Storage Type** | select | `S3` or `Azure` |
| **Path** | text | Upload path/prefix (default: `{{ workflow_run_id }}`) |

| **AWS Access Key ID** | AWS credential |
| **AWS Secret Access Key** | AWS credential |
| **Region Name** | AWS region |

</Accordion>

<Accordion title="Azure fields">

| Field | Description |
|-------|-------------|
| **Azure Storage Account Name** | Storage account name |
| **Azure Storage Account Key** | Storage account key |
| **Azure Blob Container Name** | Container name |

</Accordion>

## Communication

### Send Email

Send an email notification, optionally with file attachments from previous blocks.

| Field | Type | Description |
|-------|------|-------------|
| **Recipients** | text | Comma-separated email addresses |
| **Subject** | text | Email subject line |
| **Body** | text | Email body. Supports [parameter references](/cloud/workflow-parameters#referencing-parameters-in-blocks). |
| **File Attachments** | text | Path to files to attach |

### HTTP Request

Make an API call to an external service.

| Field | Type | Description |
|-------|------|-------------|
| **Method** | select | GET, POST, PUT, PATCH, DELETE |
| **URL** | text | Request URL. Supports parameter references. |
| **Headers** | JSON | HTTP headers |
| **Body** | JSON | Request body (POST/PUT/PATCH) |
| **Timeout** | number | Request timeout in seconds (default: 30) |
| **Follow Redirects** | boolean | Automatically follow HTTP redirects (default: true) |
| **Save Response as File** | boolean | Save the response body as a downloadable file |
| **Download Filename** | text | Filename for the saved response |

### Human Interaction

Pause the workflow and request human input. Optionally sends an email notification to reviewers. This block is created by converting a Validation block — in the block library, add a **Validation** block and use the three-dot menu to convert it to Human Interaction.

| Field | Type | Description |
|-------|------|-------------|
| **Instructions** | text | Instructions displayed to the reviewer |
| **Positive Descriptor** | text | Label for the approval action (default: "Approve") |
| **Negative Descriptor** | text | Label for the rejection action (default: "Reject") |
| **Timeout** | number | Seconds to wait before auto-continuing (default: 7200 / 2 hours) |
| **Recipients** | text | Email addresses to notify |
| **Subject** | text | Notification email subject |
| **Body** | text | Notification email body |
