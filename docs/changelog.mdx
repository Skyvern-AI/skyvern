<Update label="Week of March 30, 2026" description="Zero-config server, webhook triggers, parameter autocomplete">

## New features

- **Zero-Config Local Server** — `skyvern run server` now starts with SQLite by default, requiring no database setup. Get a fully functional local Skyvern server running in seconds without any infrastructure configuration. ([#5302](https://github.com/Skyvern-AI/skyvern/pull/5302))

- **Webhook Workflow Triggers** — Workflows can now be triggered via a dedicated webhook endpoint, enabling event-driven automations that start automatically when external systems send an HTTP request. ([#5292](https://github.com/Skyvern-AI/skyvern/pull/5292))

- **Parameter Autocomplete in Workflow Blocks** — Text inputs in workflow blocks now suggest available parameter names as you type, reducing errors and speeding up workflow authoring. ([#5293](https://github.com/Skyvern-AI/skyvern/pull/5293))

- **Test Connection for Custom Credential Services** — The custom credential service settings panel now includes a Test Connection button to verify your configuration is working before saving. ([#5299](https://github.com/Skyvern-AI/skyvern/pull/5299))

## Bug fixes

- Fixed Excel file parsing failures caused by native Python datetime values in spreadsheet cells. ([#5294](https://github.com/Skyvern-AI/skyvern/pull/5294))
- Fixed `close_page` action type missing from the action hydration mapping, which could cause errors when workflows close tabs. ([#5295](https://github.com/Skyvern-AI/skyvern/pull/5295))
- Improved browser connection error messages to be clearer and more actionable for end users. ([#5297](https://github.com/Skyvern-AI/skyvern/pull/5297))
- Fixed the parameter insert popover and AI improve button incorrectly appearing on disabled text areas. ([#5298](https://github.com/Skyvern-AI/skyvern/pull/5298))
- Fixed the Duration column being too wide, causing the Created At column to be truncated in run tables. ([#5296](https://github.com/Skyvern-AI/skyvern/pull/5296))
- Fixed gaps in failure category reporting, including unknown fallback handling and proper inheritance across workflow runs. ([#5290](https://github.com/Skyvern-AI/skyvern/pull/5290))

</Update>

<Update label="Week of March 23, 2026" description="Custom credentials, failure categories, caching improvements">

## New features

- **Custom Credential Service** — Connect Skyvern to your own credential vault via a custom HTTP endpoint. Workflows can now pull secrets from HashiCorp Vault, AWS Secrets Manager, or any internal credential store without storing them in Skyvern. ([#5240](https://github.com/Skyvern-AI/skyvern/pull/5240), [#5250](https://github.com/Skyvern-AI/skyvern/pull/5250), [#5251](https://github.com/Skyvern-AI/skyvern/pull/5251), [#5259](https://github.com/Skyvern-AI/skyvern/pull/5259))

- **Failure Categories on Tasks and Workflow Runs** — Tasks and workflow runs now include a structured `failure_category` field, making it easier to programmatically distinguish between different failure types and build targeted retry or alerting logic.

- **Full-Stack Docker Development Environment** — A Docker Compose setup for the complete Skyvern stack is now available, letting developers spin up the entire platform locally with a single command. ([#5249](https://github.com/Skyvern-AI/skyvern/pull/5249))

- **Agent-Aware CLI** — The CLI now detects TTY context, supports `--json` envelope output, and exposes a capabilities endpoint, making `skyvern` easier to embed in scripts and CI pipelines. ([#5271](https://github.com/Skyvern-AI/skyvern/pull/5271))

- **Error Codes in Task Block Validation** — Validation blocks within task flows now return structured error codes alongside failure messages, enabling more precise error handling in downstream workflow logic. ([#5265](https://github.com/Skyvern-AI/skyvern/pull/5265))

- **BotShield.js Captcha Bypass** — Added support for bypassing BotShield.js-protected pages via JavaScript token injection, expanding the range of sites Skyvern can automate reliably.

## Improvements

- Cached script execution now bundles artifacts, ensuring files generated during scripted runs are captured and available for download. ([#5247](https://github.com/Skyvern-AI/skyvern/pull/5247))
- Conditional blocks now cache script revisions per branch, improving replay accuracy when the workflow structure changes over time. ([#5248](https://github.com/Skyvern-AI/skyvern/pull/5248))
- Browser popup dialogs are no longer suppressed by default, allowing automations that need to interact with popups to proceed correctly. ([#5253](https://github.com/Skyvern-AI/skyvern/pull/5253))
- HAR files and browser console logs are now correctly attached as artifacts for task v2 workflow runs. ([#5252](https://github.com/Skyvern-AI/skyvern/pull/5252))
- Added `ENABLE_CSS_SVG_PARSING` environment variable to control SVG and CSS parsing, useful for tuning performance on workflows that don't require it. ([#5258](https://github.com/Skyvern-AI/skyvern/pull/5258))
- Added `use_in_memory_db` option to `Skyvern.local()` for faster ephemeral test and development setups. ([#5260](https://github.com/Skyvern-AI/skyvern/pull/5260))
- MCP now exposes script and caching tool endpoints, giving agents better visibility into cached execution state. ([#5243](https://github.com/Skyvern-AI/skyvern/pull/5243))

## Bug fixes

- Fixed an `AttributeError` on `Script.is_pinned` that was breaking all cached script execution. ([#5285](https://github.com/Skyvern-AI/skyvern/pull/5285))
- Fixed legacy `run_with='ai'` values causing Pydantic validation errors after the execution mode migration. ([#5286](https://github.com/Skyvern-AI/skyvern/pull/5286))
- Improved error messages when a custom credential service request fails to make failures easier to diagnose. ([#5281](https://github.com/Skyvern-AI/skyvern/pull/5281))
- Fixed TOTP parameters not being forwarded to tasks when running in cached script execution mode, restoring 2FA support for cached flows. ([#5278](https://github.com/Skyvern-AI/skyvern/pull/5278))
- Fixed cached login scripts incorrectly reporting completion when no form fields were found on the page. ([#5270](https://github.com/Skyvern-AI/skyvern/pull/5270))
- Fixed script reviewer inserting wait steps in the wrong position; waits are now correctly placed after navigation and before page interactions. ([#5268](https://github.com/Skyvern-AI/skyvern/pull/5268), [#5269](https://github.com/Skyvern-AI/skyvern/pull/5269))
- Fixed `totp_identifier` Jinja template variables not being rendered when running in cached code execution mode. ([#5263](https://github.com/Skyvern-AI/skyvern/pull/5263))
- Fixed pinned scripts being overridden by auto-corrected versions; pinned scripts now always run the exact pinned revision. ([#5261](https://github.com/Skyvern-AI/skyvern/pull/5261))
- Fixed workflows stalling indefinitely after a nested conditional block with no downstream merge point. ([#5257](https://github.com/Skyvern-AI/skyvern/pull/5257))
- Fixed Login blocks incorrectly reporting success on some pages by requiring a configurable `complete_criterion` check. ([#5256](https://github.com/Skyvern-AI/skyvern/pull/5256))
- Improved frontend security by replacing a vulnerable HTML sanitization library with DOMPurify for XSS protection.

</Update>

<Update label="v1.0.22 — February 26, 2026" description="Adaptive caching, Workflow Trigger block, CLI signup, and more">

## New features

- **Adaptive Caching (Script Generation)** — Skyvern can now learn from repeated workflow runs and generate cached scripts that speed up future executions. When a block runs successfully, Skyvern records a reusable script and replays it on subsequent runs, falling back to the AI agent only if the script fails. ([#4908](https://github.com/Skyvern-AI/skyvern/pull/4908), [#4916](https://github.com/Skyvern-AI/skyvern/pull/4916), [#4917](https://github.com/Skyvern-AI/skyvern/pull/4917), [#4920](https://github.com/Skyvern-AI/skyvern/pull/4920), [#4922](https://github.com/Skyvern-AI/skyvern/pull/4922), [#4931](https://github.com/Skyvern-AI/skyvern/pull/4931))

- **Workflow Trigger Block** — A new block type that lets one workflow trigger another from within the workflow editor. Chain workflows together as building blocks for complex automations. ([#4885](https://github.com/Skyvern-AI/skyvern/pull/4885))

- **Browser-based CLI Signup** — New users can sign up for Skyvern Cloud directly from the CLI. Running `skyvern signup` opens a browser flow that creates your account and stores your API key locally. ([#4925](https://github.com/Skyvern-AI/skyvern/pull/4925))

- **Interactive ngrok Tunnel** — `skyvern browser serve` can now create an ngrok tunnel automatically, making it easy to expose your local browser to Skyvern Cloud without manual network configuration. ([#4924](https://github.com/Skyvern-AI/skyvern/pull/4924))

- **South Korea Proxy Location** — Added `RESIDENTIAL_KR` as a proxy geolocation option for automations targeting South Korean websites. ([#4918](https://github.com/Skyvern-AI/skyvern/pull/4918))

- **Downloaded Files Tab** — Browser session detail pages now include a Downloaded Files tab for viewing and accessing files captured during a session. ([#4911](https://github.com/Skyvern-AI/skyvern/pull/4911))

- **CDP Proxy Authentication** — Remote browser connections now support authenticated proxies via the CDP `Fetch.authRequired` protocol. ([#4936](https://github.com/Skyvern-AI/skyvern/pull/4936))

- **Remote Browser Download Interception** — File downloads triggered by XHR requests and other non-navigation events are now captured when using remote CDP browser connections. ([#4906](https://github.com/Skyvern-AI/skyvern/pull/4906), [#4921](https://github.com/Skyvern-AI/skyvern/pull/4921), [#4934](https://github.com/Skyvern-AI/skyvern/pull/4934))

## Improvements

- **Renamed "Login-Free" to "Saved Profile"** — Clearer terminology throughout the UI. Browser session checkbox wording has also been improved. ([#4914](https://github.com/Skyvern-AI/skyvern/pull/4914))

- **TOTP Verification Improvements** — Better timer handling, expired code retry, 2FA banner suppression, and more reliable goal-text extraction during TOTP flows. ([#4860](https://github.com/Skyvern-AI/skyvern/pull/4860))

- **TOTP Webhook includes `workflow_permanent_id`** — Easier to identify which workflow is requesting a 2FA code. ([#4871](https://github.com/Skyvern-AI/skyvern/pull/4871))

- **Consolidated Gemini 3 Pro Model Key** — `VERTEX_GEMINI_3_PRO` now auto-resolves to the latest version, so you no longer need to update config when Google releases point versions. ([#4926](https://github.com/Skyvern-AI/skyvern/pull/4926))

- **Browser Rotation for Remote Connections** — Remote browser environments now support context rotation, improving reliability for long-running automations. ([#4929](https://github.com/Skyvern-AI/skyvern/pull/4929))

## Bug fixes

- Fixed a race condition where final task status could be overwritten by a late-arriving non-final status. ([#4928](https://github.com/Skyvern-AI/skyvern/pull/4928))
- Conditional blocks no longer crash when evaluating empty or null parameters. ([#4907](https://github.com/Skyvern-AI/skyvern/pull/4907))
- Browser sessions using `FORCE_BROWSER_SESSION` no longer time out prematurely. ([#4903](https://github.com/Skyvern-AI/skyvern/pull/4903))
- SVG-based captcha text is now correctly extracted during DOM scraping. ([#4880](https://github.com/Skyvern-AI/skyvern/pull/4880))
- MCP remote tool handlers now route to the correct API environment. ([#4869](https://github.com/Skyvern-AI/skyvern/pull/4869))
- MCP remote auth token comparison no longer fails on encrypted tokens. ([#4863](https://github.com/Skyvern-AI/skyvern/pull/4863))
- `skyvern quickstart` now handles local PostgreSQL without a pre-existing `skyvern` role. ([#4878](https://github.com/Skyvern-AI/skyvern/pull/4878))
- WebSocket URLs through ngrok tunnels now rewrite correctly for live browser streaming. ([#4927](https://github.com/Skyvern-AI/skyvern/pull/4927))

</Update>
